# プロジェクト名：乗務員向け「健康と売上の両立」支援アプリ

## 0. 本設計書の運用について
- **成長型設計書**: 本READMEは開発フェーズごとに更新される。
- **第1フェーズのスタンス**: 基盤構築を最優先とする。細部の仕様は「仮決め」を許容し、実装・検証・修正のサイクルを高速で回す。

## 1. 基本構想（中長期ビジョン）
乗務員の「健康管理」と「売上向上」を可視化・支援する。
※第1フェーズでは、これらのデータを「保存・表示できる基盤」の構築に集中する。

## 2. 技術スタック
- Framework: Expo (React Native)
- Database: SQLite (expo-sqlite)
- Version Control: GitHub

## 3. 開発チームの役割（柔軟な運用）
- **責任者**: 最終決裁。現場視点でのフィードバック。
- **設計士**: 仕様の固定と**「仮決め」の承認**。
- **実装者**: 実装・提案。**第1フェーズにおいては、必要最小限の肉付けを推奨する。**

## 4. 第1フェーズ：基盤構築仕様（仮）

### 4.1 SQLite 仮スキーマ（実装時の叩き台）
実装者は以下をベースに、型定義や制約を肉付けして良い。
- **shifts_table**: 
  - `id` (PK), `date` (TEXT/ISO8601), `type_id` (INTEGER)
- **daily_records_table**: 
  - `id` (PK), `date` (TEXT), `sales` (INTEGER), `health_score` (INTEGER), `memo` (TEXT)

### 4.2 日付管理
- ライブラリ（date-fns等）の導入を推奨。

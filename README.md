\# プロジェクト名：乗務員向け「健康と売上の両立」支援アプリ



\## 1. 基本構想

本アプリは、タクシー・バス等の乗務員が「健康を維持しながら、効率よく売上を最大化する」ことを支援するための個人向けツールである。



\### コア・バリュー

\- \*\*健康優先\*\*: 勤務サイクルと体調管理を可視化し、無理な働き方を防ぐ。

\- \*\*データ駆動の売上向上\*\*: 過去のデータに基づき、最適な稼働を支援する。

\- \*\*シンプル操作\*\*: 業務中・前後でも負担にならない直感的なUI。



\## 2. 技術スタック

\- \*\*フレームワーク\*\*: Expo (React Native)

\- \*\*データベース\*\*: SQLite (ローカル完結型)

\- \*\*開発環境\*\*: NAS上の共有フォルダ (Xドライブ等にマウント)



\## 3. 開発チームの役割と運用ルール

\- \*\*監修者（ユーザー）\*\*: 最終決定権・現場知見提供・実機動作確認。

\- \*\*設計士（Gemini）\*\*: 全体設計・仕様固定・実装指示書の作成。READMEを「唯一の正解」として管理。

\- \*\*実装者（ChatGPT）\*\*: 実装専任。READMEを絶対遵守し、独断での仕様変更を禁止する。
